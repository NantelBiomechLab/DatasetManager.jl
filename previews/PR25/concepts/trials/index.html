Trials

A Trial is a type [Julia] or class [MATLAB] that describes a single (temporal) instance of data collected from a specific subject. Trials are primarily descriptive, and do not contain the data that was collected during the trial. Trials record the (unique) subject identifier, experimental conditions (or other relevant metadata, such as subject specific characteristics), and they have have one or more Sources which refer to the files containing the actual data collected during the trial.

Julia 
julia> trial1 = Trial(1, "baseline", Dict(:group => "control", :session => 2))
Trial{Int64}
  Subject: 1
  Name: baseline
  Conditions:
    :group => "control"
    :session => 2
  No sources

MATLAB 
Trial('1', 'baseline', struct('group', 'Control', 'session', 2))

ans =

  Trial with properties:

       subject: '1'
          name: 'baseline'
    conditions: [1×1 struct]
       sources: [1×1 struct]

The descriptive nature of Trials makes it easy to include or exclude trials for an analysis based on conditions, etc. Various convenience functions have been defined for this purpose: hassubject, hascondition, and hassource.

Julia 
julia> trials = [ Trial(id, "", Dict(:group => group, :week => week, :stimulus => stim))
           for id in 1:5
           for group in 'A':'B'
           for week in 1:4
           for stim in ("sham", "low", "high") ];

julia> trials |> summary
"120-element Vector{Trial{Int64}}"
julia> trials[hascondition.(trials, :group => 'A')] |> summary
"60-element Vector{Trial{Int64}}"
julia> trials[hascondition.(trials, :group => 'A', :stimulus => ("low", "high"))] |> summary
"40-element Vector{Trial{Int64}}"
julia> filter(hassubject(3), trials) |> summary
"24-element Vector{Trial{Int64}}"

MATLAB 
trials = Trial.empty();
for id = 1:5
for group = ['A', 'B']
for week = 1:4
for stim = {'sham', 'low', 'high'}
trials(end+1) = Trial(num2str(id), '', struct('group', group, 'week', week, 'stim', stim));
end
end
end
end
